Mybatis连接Oracle数据库进行批量插入，更新

发表于 2017-08-09     阅读次数： 次
  • 在工作过程中，由于项目的特殊性（OMS系统），需要进行大量的保存，更新操作。根据公司业务需求，现在对项目进行从SqlServer到Oracle数据库的迁移。虽然只是换了数据库，但是原来的业务逻辑是不能改变的，因此参考老系统代码熟悉业务逻辑。
  • 但是，在参考老项目的时候发现了一个很严重的问题。一段保存订单信息的代码中，老系统的前辈用了mybatis配置生成的mapper文件来进行增删改查，但问题也就出在这里。一条订单信息里面包含了表头，表身，附加信息，订单产品等所有的信息。光表身里面存的就有100多条数据。调用mapper文件生成的insert语句来保存信息，需要对数据库进行频繁的获取Session，获取连接，有多少条数据就要执行多少次inset into…….语句，而且可能数据量大了还有可能出现异常，这对程序性能是有很大的影响的。
  • 所以，自己在新系统的设计上，把所有的多条数据的更新、保存全部换成了批量操作。现在直接上代码，下面再讲解用法和工作中遇到的问题。

  • 后台主要业务代码：

    • 新建list集合接收，遍历前台传过来的数据：
      List<TfPos> tfPosList = new ArrayList<TfPos>(resource.getParams().size());
 //表身的itm
 int itm = 1;
 for (TfPos params : resource.getParams()) {
     params.setPreEstDd(today);
     params.setEstDd(today);
     params.setItm(itm);
     params.setEstItm(itm);
     params.setPreItm(itm);
     params.setOsNo(resource.getOsNo());
     if(!resource.getDsptFlg().equals("1")){
         qty = params.getQty();
         params.setQty(params.getQtyVerify());
     }
     tfPosList.add(params);
     itm++;
 }
 //保存数据到表身
 commonService.save("com.ifm.mappers.operation.orderInput.tfPos", tfPosList);
    • 批量保存：
      <insert id="tfPos_insert" parameterType="list">
              INSERT INTO TF_POS (
                   OS_ID,
                   OS_NO,
                   ITM,
                   PRD_NO,
                   PRD_NAME,
                   QTY,
                   QTY_VERIFY
                  )
      <foreach collection="list" item="item" index="index" separator="UNION ALL" >
                select
                    'SO',
                    #{item.osNo},
                    #{item.itm},
                    #{item.prdNo},
                    #{item.prdName},
                    #{item.qty},
                    #{item.qtyVerify}
                from dual
        </foreach>
</insert>
    • 前台传过来一共2356条数据，把这些封装在list里面，批量插入打印的log日志查看，用时不到0.3秒。

    • 批量更新和插入一样，这里就不贴业务代码了，只看一下sql：

      <update id="orderAttach_update"  parameterType="list">  
             <foreach collection="list" item="item" index="index" open="begin" close=";end;" separator=";">
                        update GKHT_DSPT_ORDERATTACH t 
                             set 
                                t.ATTACHVALUE=#{item.attachValue,jdbcType=VARCHAR}
                             where t.ORDERCODE = #{item.osNo,jdbcType=VARCHAR}
                             and t.ATTACHNAME= #{item.attachName,jdbcType=VARCHAR}
            </foreach>
</update>

    • foreach主要用在构建in条件中，它可以在SQL语句中进行迭代一个集合。

    • foreach元素的属性主要有 item，index，collection，open，separator，close。
      1 . item表示集合中（代码里list集合里封装bean：TfPos）每一个元素进行迭代时的别名，
      2 . index指定一个名字，用于表示在迭代过程中，每次迭代到的位置，
      3 . open表示该语句以什么开始，
      4 . separator表示在每次进行迭代之间以什么符号作为分隔符，
      5 . close表示以什么结束，

    • 在使用foreach的时候最关键的也是最容易出错的就是collection属性，该属性是必须指定的，但是在不同情况下，该属性的值是不一样的，主要有一下3种情况：
      1 .如果传入的是单参数且参数类型是一个List的时候，collection属性值为list
      2 .如果传入的是单参数且参数类型是一个array数组的时候，collection的属性值为array
      3 .如果传入的参数是多个的时候，我们就需要把它们封装成一个Map了，当然单参数也可以封装成map

  • 运行通过。在Oracle的版本中，有几点需要注意的：
    1 .SQL中没有VALUES；
    2 .标签中的(selece ….. from dual)，oracle专有；
    3 .标签中的separator的属性为”UNION ALL”，将查询合并结果集。
