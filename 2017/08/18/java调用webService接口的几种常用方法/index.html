java调用webService接口的几种常用方法

发表于 2017-08-18     阅读次数： 次
* 需求分析：由于系统改进，需要和其他公司多系统进行联调开发，系统之间的数据交互通过调用彼此的webService接口，webService主要解决了不同的系统或者调用分布部署的数据处理问题

  • 现在总结一下，在开发过程中常用的三种调用接口方式，废话少说，直接撸代码，简单粗暴，哈哈：
    1 .采用http协议发送请求

    • 使用过webservice的童鞋大概都明白它是基于Soap协议交换数据的，同时Soap协议是对HTTP协议的扩展，其实我们就可以认为调用一个WEB服务就是通过http协议GET或POST数据的过程，只不过中间的输入/输出数据是遵守Soap协议格式的标准XML。明白这个道理之后我们就可以通过构造请求数据来模拟调用WEB服务的过程了

      public static String doPost(String url,String method, String params) throws     IOException {  
   URL urlCon=new URL(url);
   //建立http连接
   HttpURLConnection    conn = (HttpURLConnection) urlCon.openConnection();
   //是否具有输入参数
   conn.setDoInput(true);
   //是否输出输入参数
   conn.setDoOutput(true);
   //发POST请求
   conn.setRequestMethod("POST");
   //设置请求头（注意一定是xml格式）
   conn.setRequestProperty("content-type", "text/xml;charset=utf-8");
   //设置用户名密码，esb认证
   String username = "esb_oms01";
   String password = "gkht_7890";
   String input = username + ":" + password; 
   //对其进行加密
   byte[] base64 = Base64.encodeBase64(input.getBytes());          
   String strBase64 = new String(base64);
   //httpBasic认证
   String authorization = "Basic "+strBase64;
   conn.setRequestProperty( "Authorization",authorization); 
   // 构造请求体，符合SOAP规范（最重要的）
   String requestBody = "<soapenv:Envelope xmlns:soapenv=\"http://schemas.xmlsoap.org/soap/envelope/\" xmlns:axis=\"http://ws.apache.org/axis2\">"
       + "<soapenv:Header/>"
       + "<soapenv:Body>"
       + "<axis:"+method+">" //你要访问的接口方法
       +"<!--Optional:-->"
       + "<sData>"
       + params                 //参数
       + "</sData>"
       + "</axis:"+method+">"//你要访问的接口方法
       + "</soapenv:Body>"
       + "</soapenv:Envelope>";

   //获得一个输出流
   OutputStream out = conn.getOutputStream();
   out.write(requestBody.getBytes());
   System.out.println(requestBody);
   //获得服务端响应状态码
   int code = conn.getResponseCode();
   StringBuffer sb = new StringBuffer();
   if(code == 200){
       //获得一个输入流，读取服务端响应的数据
       InputStream is = conn.getInputStream();
       byte[] b = new byte[1024];
       int len = 0;
       while((len = is.read(b)) != -1){
       String s = new String(b,0,len,"utf-8");
       sb.append(s);
   }
   is.close();
   }else {
   System.out.println("接口调用失败"+code);
   }
   //关闭连接
   out.close();
   System.out.println("服务端响应数据为："+sb.toString());
  //定义返回结果
   String result=sb.toString();
   return result;
   }

    2 .采用axis2来调用

    • Axis2是一套崭新的WebService引擎，该版本是对Axis1.x重新设计的产物。Axis2不仅支持SOAP1.1和SOAP1.2，还集成了非常流行的REST WebService，同时还支持Spring、JSON等技术

      //使用axis2调用ebs工具接口
try {  
        ServiceClient sc = new ServiceClient();  
        Options opts = sc.getOptions();  
        opts.setTo(new EndpointReference(esbUrl+"/SalesSB/SaleOrder/Gjgl/ProxyServices/SalesGjglToolOrderSyncSoapProxy"));  
        opts.setAction("urn:ToolsOutNotice");  
        HttpTransportProperties.Authenticator basicAuth = new HttpTransportProperties.Authenticator();  
        basicAuth.setUsername("esb_oms01");  
        basicAuth.setPassword("gkht_7890");  
        opts.setProperty(HTTPConstants.AUTHENTICATE, basicAuth);  
        OMElement res = sc.sendReceive(createPayLoad(params));

}
